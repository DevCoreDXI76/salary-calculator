# 프로젝트: 샐러리핏(SalaryFit) 수익화 그로스 프로젝트

> 작성일: 2026-07-22
> 대상: https://salaryfit.coredxi.com (연봉 실수령액 계산기 및 부가 도구 모음)
> 목적: 애드센스 광고 수익 창출 + 트래픽 성장을 동시에 추진하는 2트랙 그로스 프로젝트 킥오프 문서

---

## 1. 현황 진단 (연결된 저장소 + 실사이트 점검 결과)

레포지토리(`docs/prd.md`, `spec.md`, `plan.md`, `adsense-checklist.md`, `ops-maintenance.md`)와 실제 배포 사이트를 직접 확인한 결과는 다음과 같다.

**잘 되어 있는 것**
- React 19 + Vite + Tailwind 기반으로 4개 계산기(연봉/퇴직금/실업급여/연차)가 실제로 정상 동작한다. 연봉 40,000,000원 입력 시 실수령액이 즉시 계산되는 것을 화면에서 확인했다.
- `robots.txt`, `sitemap.xml`, 개인정보처리방침, `ads.txt`가 모두 배포되어 있다. sitemap에는 계산기 4개 페이지 외에 팁 아티클 10개(`/tips/salary-negotiation-basics` 등)가 등록되어 있어 SEO용 콘텐츠 뼈대는 이미 갖춰진 상태다.
- `public/ads.txt`에 플레이스홀더가 아니라 **실제 게시자 ID(`pub-8125995278513075`)** 가 이미 들어가 있다. 이는 애드센스 사이트 등록/검증 절차를 최소 한 번은 진행했다는 뜻이다.
- GA4가 연결되어 실제 트래픽 데이터를 확인할 수 있는 상태(사용자 확인).

**핵심 문제 — 왜 광고가 안 붙는지**
- 실제 배포된 `index.html`을 확인한 결과, **애드센스 스크립트 태그(`adsbygoogle.js`)가 코드에 전혀 삽입되어 있지 않다.** `docs/adsense-checklist.md`에도 이 단계가 아직 "승인 후 (필요 시) 추가"로 남아 있어, 코드 관점에서는 심사 통과 여부와 무관하게 지금 광고가 노출될 수 없는 구조다.
- 즉, 애드센스 계정 승인 상태와 별개로 **① 계정 승인 여부 확인 → ② 승인됐다면 `VITE_ADSENSE_CLIENT` 환경변수 설정 + 스크립트 삽입 + 재배포**라는 명확히 막힌 지점이 하나 있다.

**콘텐츠/구조 관점에서 아쉬운 점 (조회수가 안 느는 이유 후보)**
- 계산기 4종 + 팁 아티클 10편 정도로, 도구형 사이트치고는 첫 시작치고 나쁘지 않지만 검색 유입을 크게 늘리기엔 콘텐츠 양·키워드 커버리지가 아직 얕다.
- `companySalaries.js`(회사별 연봉 비교로 추정)처럼 준비된 데이터가 있는데 실제 내비게이션에는 노출되지 않는 등, 이미 만들어둔 자산이 다 활용되지 않고 있을 가능성이 있다.
- "이 돈으로 뭐 살까" 같은 재미 요소(PRD에 명시)가 실제 화면에서 확인되지 않아, 체류시간을 늘리는 장치가 계획 대비 부족해 보인다.

## 2. 프로젝트 목표 (Goals)

사용자가 우선순위를 "기술적 수익화 고치기 + 트래픽 확대"를 **동시에** 진행하는 것으로 정했으므로, 두 트랙을 병렬로 두되 순서에 의존관계가 있는 항목은 순차 배치한다.

1. **[트랙 A] 수익화 정상화** — 이미 승인/등록된 애드센스 자산(ads.txt, 사이트 정책 페이지)을 실제 광고 노출로 연결한다. 목표: 4주 내 광고가 실제로 화면에 노출되고 노출수/클릭수가 애드센스 콘솔에서 확인되는 상태.
2. **[트랙 B] 트래픽 성장** — SEO 콘텐츠와 신규 계산기/비교 콘텐츠를 확장해 검색 유입을 늘린다. 목표: 3개월 내 GA4 기준 오가닉 세션이 의미 있게(예: 2~3배) 증가.
3. **[공통 원칙] 지속 가능한 운영 체계** — `ops-maintenance.md`에 이미 있는 연간 요율 갱신 루틴처럼, 주 5~10시간 혼자 운영 가능한 루틴으로 관리한다. 화려한 신규 기능보다 "검색엔진에 걸리는 콘텐츠 + 실제로 켜지는 광고"를 우선한다.

## 3. 프로젝트 지침 (진행 원칙)

- **막힌 것부터 뚫는다.** 트래픽을 아무리 늘려도 광고 스크립트가 없으면 수익이 0이다. Phase 0(광고 활성화)은 다른 작업과 무관하게 최우선으로 먼저 끝낸다.
- **이미 만든 자산을 먼저 확인한다.** `companySalaries.js`, `financialNews.js`, `financialTips.js` 등 이미 존재하지만 노출이 안 된 콘텐츠/데이터를 먼저 점검해 "새로 안 만들어도 되는 일"을 줄인다.
- **AdSense 정책 위반 리스크를 항상 우선 점검한다.** 클릭 유도 문구, 광고와 콘텐츠 혼동 UI는 심사·계정 정지 사유가 되므로 신규 기능을 추가할 때마다 `docs/adsense-checklist.md` 기준으로 재확인한다.
- **주 단위로 작게 배포한다.** 주 5~10시간 혼자 진행이므로, 한 번에 큰 리뷰를 하지 않고 Phase 내 항목을 1~2개씩 끝내고 바로 배포 → 확인하는 사이클을 반복한다.
- **측정 없이 다음 단계로 안 넘어간다.** GA4·Search Console·애드센스 콘솔 수치를 매주 한 번은 확인하고, 이 문서의 로드맵을 그 수치에 맞춰 조정한다.

## 4. 로드맵

### Phase 0 — 광고 활성화 (최우선, 목표: 1주 이내)
- [ ] 애드센스 콘솔에 로그인해 사이트 심사/계정 상태를 직접 확인 (승인/대기/반려 중 어떤 상태인지)
- [ ] 승인된 상태라면 `VITE_ADSENSE_CLIENT`(`ca-pub-...`) 값을 Vercel 프로덕션 환경변수에 등록
- [ ] `index.html`에 애드센스 스크립트 태그 삽입 (`docs/adsense-checklist.md` 4번 항목 참고)
- [ ] 광고 슬롯(`AdSlot.jsx`)이 실제 `<ins class="adsbygoogle">` 단위와 연결되도록 연동
- [ ] 재배포 후 실제 화면에 광고가 노출되는지, 정책 위반 경고가 뜨지 않는지 확인
- [ ] 반려/대기 상태라면 반려 사유를 확인해 그에 맞는 보완(정책 페이지, 콘텐츠 등) 진행

### Phase 1 — 숨은 자산 점검 및 노출 (1~2주)
- [ ] `companySalaries.js` 데이터가 실제 화면 어디에도 안 쓰이고 있는지 확인하고, 쓰인다면 내비게이션에서 찾기 쉽게 노출
- [ ] PRD에 명시된 "이 돈으로 뭐 살까" 소비 시뮬레이터(`Consumption.jsx`)가 실제 배포본에 보이는지 확인, 안 보이면 노출 위치 점검
- [ ] `financialNews.js`, `financialTips.js` 10편이 팁 목록 페이지(`/tips`)에서 잘 탐색되는지, 사이트 내부 링크로 서로 연결되는지 확인
- [ ] GA4에서 현재까지의 실제 페이지별 조회수/체류시간/이탈률 데이터를 확인해 어떤 페이지가 유입은 있는데 광고 노출/체류시간이 낮은지 파악

### Phase 2 — SEO 콘텐츠 및 신규 계산기 확장 (2~6주)
- [ ] Search Console에서 현재 색인된 페이지, 검색 노출 키워드, 클릭률을 확인해 확장할 키워드 우선순위 결정
- [ ] 팁 아티클을 계절/이슈 연계(연말정산, 최저임금 발표 시즌 등) 주제로 추가
- [ ] 검색 수요가 있는 인접 계산기(예: 프리랜서 종합소득세, 아르바이트 주휴수당, 4대보험 모의계산 등) 중 1~2개를 신규 기능으로 추가
- [ ] 계산기 결과 페이지에 관련 팁 아티클을 자동 추천해 페이지뷰(PV)를 늘리는 내부 링크 구조 보강

### Phase 3 — 리텐션 및 수익 최적화 (지속)
- [ ] 애드센스 콘솔의 실제 수익/CTR 데이터를 바탕으로 광고 슬롯 위치·개수 A/B 조정
- [ ] 재방문을 유도할 요소(최근 계산 기록 활용, 연봉 협상 시즌 알림성 콘텐츠 등) 검토
- [ ] `ops-maintenance.md`의 연 1~2회 요율 갱신 루틴에 맞춰 매년 초 세율/보험료율 업데이트 스케줄링
